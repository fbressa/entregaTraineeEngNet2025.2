# Matriz de rastreabilidade

O objetivo de uma matriz de rastreabilidade é mapear a relação entre diferentes artefatos do projeto.


**Legenda:**


**✔** = **Rastreabilidade Direta:** O Requisito Funcional (RF) implementa diretamente a User Story (US).

**○** = **Rastreabilidade Indireta:** O RF é relacionado, necessário como suporte ou é um fluxo secundário da US.

**✘** = **Sem Rastreabilidade:** O RF não está relacionado a esta US.
